# 매매 전략 구조

이 디렉토리는 두 가지 타입의 전략을 포함합니다:
1. **백테스트 전략** (`BacktestStrategy` 상속) - 과거 데이터로 전략 성능 검증 (동기)
2. **실시간 거래 전략** (`TradingStrategy` 상속) - 실시간 매매 신호 생성 (비동기)

## 파일 구조

```
base_strategy.py                    # 백테스트 전략 베이스
├── ema_strategy.py                 # EMA 골든크로스 백테스트
├── ichimoku_strategy.py            # 일목균형표 백테스트
└── single_ema_backtest_strategy.py # 단일 20EMA 백테스트

base_trading_strategy.py            # 실시간 거래 전략 베이스
base_single_ema.py                  # 실전/백테스트 공통 파라미터 및 trailing stop 파라미터
└── single_ema_strategy.py          # 단일 20EMA 실전 전략

trading_strategy_factory.py         # 실시간 전략 팩토리
strategy_factory.py                 # 백테스트 전략 팩토리 (상위 디렉토리)
```

## 단일 20EMA 실전 전략 (SingleEMAStrategy)

20일 지수이동평균(EMA)을 중심으로 추세 추종 매매를 수행하는 전략입니다.
분할 매수/매도를 지원하며, 장중 실시간 신호 기반으로 매매를 실행합니다.

### 사용 지표

| 지표 | 용도 |
|-----|------|
| EMA 20 | 단기 추세 판단, 매수/매도 기준선 |
| ADX / +DI / -DI | 추세 강도 및 방향 판단 |
| ATR (14) | 변동성 기반 손절선, 가격 가드레일 |
| OBV z-score | 거래량 기반 수급 강도 |

모든 지표는 전일 종가 기준으로 Redis에 캐시되며, 장중에 현재가를 이용한 증분 계산으로 실시간 업데이트됩니다.

### 배치 스케줄

| 시간 | 배치 | 설명 |
|-----|------|------|
| 08:29 | `cache_warmup_job` | 지표 캐시 워밍업 (Redis) |
| 09:00~14:55 | `trade_job` | 장중 매수/매도 체크 (5분 간격) |
| 15:35 | `day_collect_job` | 일별 데이터 수집 |

### SIGNAL 상태 흐름

```
SIGNAL 0 (매수 대기)
  │ 매수 (Conviction 기반 수량 계산)
  ▼
SIGNAL 1 (보유 - 1차 익절 전) ← PEAK_PRICE = 매수가
  │ 장중 5분마다: PEAK_PRICE = max(PEAK_PRICE, 현재가)
  ├─ [손절] EMA-ATR×2.0 ──────→ OBV z ≤ 0? → SIGNAL 0
  │                              OBV z > 0? → SIGNAL 3
  └─ [1차 익절] PEAK-ATR×2.0 → 50% 매도 → SIGNAL 2
                                              │ PEAK_PRICE 리셋 (현재가)
                                              │ 장중 5분마다: PEAK_PRICE = max(PEAK_PRICE, 현재가)
                                              ├─ [손절] Max(EMA-ATR×2.0, 평단가) → OBV z 분기 → SIGNAL 0 or 3
                                              └─ [2차 익절] PEAK-ATR×2.0 AND OBV-Z14<-0.5
                                                  → OBV z 분기 → SIGNAL 0 or 3
                                                  (OBV 양호 시 → 추세 추적 계속)

SIGNAL 3 (수급 이탈 대기)
  │ OBV z-score < COOLDOWN_OBV_EXIT 확인 (수급 이탈 완료)
  ▼
SIGNAL 4 (수급 재유입 대기)
  │ OBV z-score > COOLDOWN_OBV_REENTRY 확인 (수급 재유입 확인)
  ▼
SIGNAL 0 (매수 대기)
```

### 포지션 사이징 (Conviction Score 기반)

매수 신호의 강도(Conviction Score)에 따라 투입 비율을 가변 조절합니다.
강한 신호일수록 더 많이 투입하고, 애매한 신호일수록 줄입니다.

```
Qty = int(CUR_AMOUNT × MAX_ENTRY_PCT × Conviction / 현재가)

MAX_ENTRY_PCT = 0.8 (최대 배정금의 80%)
Conviction = 0.4 ~ 1.0 (신호 강도에 따라 가변)
```

#### Conviction Score 계산

OBV z-score(70%)와 ADX(30%)의 가중 평균으로 산출합니다.

| 요소 | 가중치 | 점수 매핑 | 근거 |
|------|--------|----------|------|
| OBV z-score | 70% | z=0→0.3, z=1.0→0.77, z=1.5+→1.0 | 수급 강도 → 추세 지속 가능성 |
| ADX | 30% | 15→0.3, 25→1.0, 30+→0.8 (과열 감점) | 추세 강도 (과열 시 감점) |

#### 신호 강도별 투입 예시

| 신호 강도 | OBV z | ADX | Conviction | 투입비 |
|----------|-------|-----|-----------|--------|
| 약한 신호 (겨우 통과) | 0.1 | 16 | 0.40 | 32% |
| 보통 신호 | 0.5 | 20 | 0.57 | 46% |
| 강한 신호 | 1.0 | 24 | 0.82 | 66% |
| 매우 강한 신호 | 1.5 | 25 | 1.00 | **80%** |

#### CUR_AMOUNT 변동

종목별 투자 가능 금액은 매매 체결 시 자동으로 변동됩니다.

- **초기**: `CUR_AMOUNT = INIT_AMOUNT` (사용자가 설정한 초기 투자금)
- **매수 체결 시**: `CUR_AMOUNT -= 체결금액` (가용 금액 감소)
- **매도 체결 시**: `CUR_AMOUNT += 체결금액` (가용 금액 증가)

수익이 누적되면 CUR_AMOUNT가 커져 다음 매수 시 더 많은 수량을 매수하고,
손실이 누적되면 줄어들어 자연스럽게 리스크가 축소됩니다.

### 매수 전략

공통 필터 + **시나리오 A 또는 B** 충족 후 **연속 2회(약 10분)** 확인 시 진입 (현재가 기준)

**공통 필터:**
- **급등 필터**: 당일 변동률(절대값) ≤ 5% (급등/급락 종목 배제)
- **윗꼬리 필터**: 전일 캔들 범위 > 종가×3% AND 윗꼬리 비율 ≥ 40% 시 매수 차단
- **갭 하락 필터**: 현재가 < 전일 저가 시 매수 차단

**시나리오 A — 눌림목 매집 진입**
- **가격 범위**: 실시간 EMA20 ± ATR×0.5 이내
- **수급 개선**: OBV z-score > 0 AND 전일 대비 상승
- **추세 존재**: ADX 18~30 + +DI > -DI

**시나리오 B — 추세 추종 돌파 진입**
- **가격 확인**: 현재가 > 실시간 EMA20 (괴리율 3% 이내)
- **추세 방향**: +DI > -DI + ADX > 15
- **수급 동반**: OBV z-score > 0

> 연속 2회 조건은 Redis에 상태를 저장하여 swing_id별로 독립 관리합니다.

### 매도 전략

#### [손절] 장중 즉시 매도 (5분마다 체크)

- **SIGNAL 1**: 현재가 ≤ EMA20 - ATR×2.0 시 전량 매도
- **SIGNAL 2**: 현재가 ≤ Max(EMA20 - ATR×2.0, 평단가) 시 전량 매도 (**본전 방어**)
- ATR=0인 경우 손절 체크 스킵 (HOLD)

#### [PEAK 추적] 현재가 기반 (노이즈 방지)

- PEAK_PRICE = max(PEAK_PRICE, **현재가**) — 장중고가 대신 5분 체크 시점의 현재가 사용
- 순간 스파이크(시초가 오버슈팅, 장중 노이즈)가 PEAK를 오염시키지 않음
- 5분 이상 유지되지 않는 가격은 PEAK에 반영되지 않아 가짜 익절 신호 방지

#### [1차 익절] 50% 매도 (SIGNAL 1 → 2)

- 현재가 ≤ 고점(PEAK_PRICE) - ATR×2.0
- 보유 물량의 **50%** 즉시 매도하여 수익 확정
- 1차 익절 후 PEAK_PRICE를 현재가로 리셋 (2차 추적용)

#### [2차 익절] 잔량 전량 매도 (SIGNAL 2 → 0 or 3)

- 현재가 ≤ 고점(PEAK_PRICE) - ATR×2.0 **AND** OBV z-score(14일) < -0.5
- 수급(OBV 14일)이 살아있으면 ATR 조건 충족해도 **매도 안 함** → 대시세 추적
- OBV 꺾이면 잔량 전량 매도 → OBV z(7일) 상태에 따라 SIGNAL 0 or 3

### 체결 분할 (TWAP)

대량 주문 시 시장 충격(슬리피지)을 줄이기 위해 주문을 자동으로 여러 사이클에 나눠 체결합니다.
매수/매도 모두 동일하게 적용됩니다.

#### 발동 조건

- 주문금액(또는 목표수량 × 현재가)이 **일평균거래대금 × 0.5%** 초과 시 자동 분할
- 이하이면 단일 주문으로 한 번에 처리

#### 분할 실행 흐름

```
1사이클 (신호 발생 시점)
    │
    ├─ 단일 주문 가능 → 즉시 전량 체결 (completed: True)
    │
    └─ 분할 필요 → 첫 chunk 주문 + Redis에 진행 상태 저장 (completed: False)
                        │
                        ├─ 다음 사이클 (5분 후)
                        │   └─ continue_partial_execution 호출
                        │       ├─ 매수 중 손절 신호 발생 → 매수 중단 (aborted: True)
                        │       ├─ 잔여 chunk 주문 + 상태 업데이트
                        │       └─ 완료 시 Redis 키 삭제 (completed: True)
                        │
                        └─ 목표 금액/수량 소진까지 반복
```

#### 우선순위

분할 체결이 진행 중(`partial_exec:{swing_id}` Redis 키 존재)이면 신호 로직보다 **최우선으로 처리**됩니다.
분할이 완전히 완료된 후 SIGNAL 상태가 업데이트됩니다.

#### 매수 중 손절 처리

분할 매수 진행 중 EMA-ATR 손절 조건 충족 시:
- 매수 즉시 중단 (추가 chunk 취소)
- 이미 매수한 수량은 보유 유지
- 보유 수량 있으면 SIGNAL 1, 없으면 SIGNAL 0으로 처리

| 파라미터 | 값 | 설명 |
|---------|-----|------|
| SLIPPAGE_RATIO | 0.005 | 사이클당 일평균거래대금 비율 (0.5%) |

## 실전 전략 vs 백테스팅 전략 비교

| 구분 | 실전 전략 (SingleEMAStrategy) | 백테스팅 전략 (SingleEMABacktestStrategy) |
|------|-------------------------------|------------------------------------------|
| **데이터** | 실시간 5분 간격 체크 | 일별 OHLCV 데이터 |
| **매수 공통 필터** | 급등 + 윗꼬리 + 갭하락 | 동일 |
| **매수 시나리오** | A(눌림목 매집) + B(추세 추종 돌파) | 동일 |
| **연속 확인** | 2회 (Redis, ~10분) | 1회 (일봉 한계) |
| **포지션 사이징** | 배정금×MAX_ENTRY_PCT×Conviction (실시간 ADX/OBV) | 동일 (전일 종가 기준 ADX/OBV) |
| **PEAK 추적** | max(PEAK, 현재가) — 5분마다 | max(PEAK, 당일고가) — 매도 전 갱신 |
| **손절** | 동적 EMA-ATR (매 체크 재계산) | 고정 손절가 (매수 시 확정, 일봉 한계) |
| **손절 후 SIGNAL** | OBV z 판단 → SIGNAL 0 or 3 | 항상 SIGNAL 3 (보수적) |
| **1차 익절** | PEAK - ATR×2.0 → 50% 매도 | 동일 (체결가 < 매수가면 스킵) |
| **2차 익절** | PEAK - ATR×2.0 + OBV-Z14 < -0.5 | 동일 |
| **수급 안정화** | SIGNAL 3→4→0 (2단계) | 동일 |
| **포지션 상태** | SIGNAL 0→1→2→3→4→0 | 동일 |
| **주요 목적** | 장중 실시간 매매 | 과거 데이터 성능 검증 |

### 데이터 해상도에 따른 의도적 차이 (6건)

전략 로직 자체는 동일하지만, 실전(5분 간격)과 백테스팅(일봉)의 데이터 해상도 차이로 인해 구현 방식이 다른 항목입니다. **전략 변경이 아닌 데이터 한계에 대한 보정**이므로, 한쪽을 수정할 때 다른 쪽을 동일하게 바꾸면 안 됩니다.

| # | 항목 | 실전 | 백테스팅 | 차이 이유 |
|---|------|------|---------|----------|
| 1 | **연속 확인** | 2회 (10분) | 1회 | 일봉은 장중 반복 체크가 불가능하므로 1회로 충분 |
| 2 | **PEAK 소스** | `현재가` (5분 샘플링) | `당일 고가` | 실전은 5분마다 현재가를 샘플링하여 노이즈 필터링. 백테스팅은 장중 현재가가 없으므로 일봉 고가 사용 |
| 3 | **손절 기준** | 동적 `EMA - ATR` (매 체크 재계산) | 고정 `매수가 - ATR` (매수 시 확정) | 실전은 매 5분 EMA/ATR이 변하므로 동적 계산. 백테스팅은 장중 EMA 변화를 추적할 수 없어 매수 시점 값으로 고정 |
| 4 | **손절 후 SIGNAL** | OBV z 판단 → 0 or 3 | 항상 3 | 실전은 매도 시점에 실시간 OBV z를 알 수 있어 즉시 분기 가능. 백테스팅은 보수적으로 항상 수급 안정화를 거침 |
| 5 | **갭 하락 필터** | `현재가 < 전일 저가` | `당일 시가 < 전일 저가` | 실전은 매 체크 시점 현재가가 전일 저가 아래인지 확인 (갭+장중 하락 모두 차단). 백테스팅은 시가만 알 수 있어 갭 하락만 감지 |
| 6 | **1차 익절 수익 체크** | 없음 | 체결가 < 매수가면 익절 스킵 | 실전은 실시간 가격으로 체크하므로 갭하락과 익절이 동시에 걸리지 않음. 백테스팅은 일봉에서 갭하락 시 익절 체결가가 손실가가 될 수 있어 보정 필요 |
